# 有給休暇申請機能 — ユーザーストーリー

## 概要

「申請 → 承認 → 反映」のワークフローを持つ有給休暇申請機能。
勤怠修正（correction）機能と同じ承認フローパターンを踏襲する。

---

## 休暇種別

| 種別 | 説明 |
| --- | --- |
| 年次有給休暇 | 通常の有給休暇 |
| 特別休暇 | 慶弔休暇、夏季休暇等 |

- 半日休暇（午前休・午後休）に対応する
- 残日数管理・付与ルールはスコープ外（初期データで固定 20 日）

---

## 有給休暇申請

| ID | ストーリー | ロール |
| --- | --- | --- |
| LEAVE-01 | 休暇種別・取得日（開始日〜終了日）・全日/午前休/午後休を指定して有給休暇を申請できる | 全ロール |
| LEAVE-02 | 連続日数（例: 7/14〜7/18）を1回の申請で出せる | 全ロール |
| LEAVE-03 | 理由は任意で入力できる | 全ロール |
| LEAVE-04 | 過去日の申請はできない（エラーになる） | 全ロール |
| LEAVE-05 | 自分が出した休暇申請の状態（申請中/承認/却下）を確認できる | 全ロール |
| LEAVE-06 | 自部署メンバーの休暇申請を一覧で確認できる | 上長 |
| LEAVE-07 | 自部署メンバーの休暇申請を承認または却下できる | 上長 |
| LEAVE-08 | 承認された休暇は自動的に勤怠データに反映される（有給フラグが立つ） | システム |
| LEAVE-09 | 有給取得日に打刻があっても受け付ける（半日休で午後出勤等） | システム |
| LEAVE-10 | 月次集計に「有給取得日数」が含まれる | システム |

---

## 承認フロー

勤怠修正と同一:

| 申請者 | 承認者 |
| --- | --- |
| 一般社員 | 所属部署の上長 |
| 上長 | 自己承認 |
| 管理者（上長でない） | 所属部署の上長 |
| 管理者（上長を兼任） | 自己承認 |

- 取り下げ機能はスコープ外

---

## 半日休暇の定義

| 区分 | 時間帯 | 勤怠への影響 |
| --- | --- | --- |
| 全日休 | 終日 | 打刻なしでも欠勤にならない |
| 午前休 | 午前のみ休暇 | 午後の打刻を受け付ける |
| 午後休 | 午後のみ休暇 | 午前の打刻を受け付ける |

---

## 勤怠データへの反映（LEAVE-08 詳細）

- 承認時、対象日の勤怠レコードに「有給」フラグを立てる
- 有給フラグが立った日は打刻なしでも欠勤日数にカウントしない
- 半日休の場合、打刻があればその時間を勤務時間として計上する

---

## 月次集計への影響（LEAVE-10 詳細）

月次集計に以下の項目を追加:

- 有給取得日数（全日休 = 1日、半日休 = 0.5日で計算）

---

## 楽観ロック

勤怠修正と同様に `version` フィールドによる楽観ロックを使用する。
承認/却下の競合時は後の操作が HTTP 409 エラーになる。

---

## 初期データ

- 各社員に有給残日数 20 日を設定（参考値。残日数バリデーションはしない）

---

## スコープ外

- 有給残日数の管理（超過チェック）
- 有給付与ルール（勤続年数に応じた自動付与）
- 申請の取り下げ
- 代理承認
